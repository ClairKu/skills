# 需求价值分析

## 快速开始

收到需求分析请求时：

1. **理解需求背景** - 确认需求来源、目标用户、业务目标
2. **选择分析框架** - 根据场景选择合适的框架
3. **收集评估数据** - 获取评估所需的关键信息
4. **执行价值分析** - 应用框架进行量化分析
5. **输出分析结果** - 生成报告、评分卡或矩阵

## 框架选择指南

| 场景 | 推荐框架 | 适用说明 |
|------|----------|----------|
| 功能优先级排序 | RICE | 需要量化对比多个功能 |
| 版本规划 | MoSCoW | 快速分类必要/非必要功能 |
| 用户体验优化 | KANO | 理解用户满意度驱动因素 |
| 投资决策 | ROI + RICE | 需要成本收益分析 |
| 综合评估 | 多框架组合 | 复杂需求需多维度分析 |

## 分析工作流

复制此清单跟踪进度：

```
分析进度：
- [ ] 步骤1：需求澄清
- [ ] 步骤2：框架选择
- [ ] 步骤3：数据收集
- [ ] 步骤4：价值评估
- [ ] 步骤5：输出结果
```

### 步骤1：需求澄清

向用户确认以下信息：

- **需求描述**：具体要实现什么功能/解决什么问题？
- **目标用户**：谁会使用这个功能？用户规模多大？
- **业务目标**：这个需求支撑什么业务目标？
- **约束条件**：时间、资源、技术有什么限制？
- **成功指标**：如何衡量这个需求的成功？

### 步骤2：框架选择

**单需求深度分析** → KANO + ROI  
**多需求排序** → RICE  
**版本范围确定** → MoSCoW  
**不确定时** → 先用 RICE 量化，再用 MoSCoW 分类

### 步骤3：数据收集

根据选择的框架收集数据，参见 [frameworks.md](frameworks.md)

### 步骤4：价值评估

应用框架公式进行计算：

**RICE 评分** = (Reach × Impact × Confidence) / Effort

**ROI** = (预期收益 - 成本) / 成本 × 100%

### 步骤5：输出结果

根据需要生成：结构化报告、评分卡或优先级矩阵，参见 [templates.md](templates.md)

## RICE 快速评分参考

| 维度 | 评分标准 |
|------|----------|
| **Reach（触达）** | 每月/季度影响的用户数 |
| **Impact（影响）** | 3=巨大 2=高 1=中 0.5=低 0.25=微小 |
| **Confidence（信心）** | 100%=高 80%=中 50%=低 |
| **Effort（工作量）** | 人月数（0.5, 1, 2, 3...） |

## MoSCoW 分类标准

| 类别 | 含义 | 判断依据 |
|------|------|----------|
| **Must** | 必须有 | 没有则产品无法发布/使用 |
| **Should** | 应该有 | 重要但可延期，有临时方案 |
| **Could** | 可以有 | 锦上添花，资源允许时做 |
| **Won't** | 不做 | 本期不做，可能未来做 |

## KANO 分类标准

| 类型 | 特征 | 策略 |
|------|------|------|
| **基本型** | 有了不满意度降低，没有强烈不满 | 必须满足 |
| **期望型** | 有了满意度提升，没有满意度下降 | 尽量满足 |
| **魅力型** | 有了惊喜，没有也不会不满 | 差异化竞争点 |
| **无差异型** | 有没有都一样 | 不投入 |
| **反向型** | 有了反而不满 | 避免 |

## 常见问题处理

**数据不足时**：使用保守估计，Confidence 设为 50%

**多个需求对比时**：统一框架和评分标准后再对比

**需求不明确时**：先澄清需求，再进行分析
