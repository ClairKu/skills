## 优先级矩阵模板

```markdown
# 优先级矩阵

## 价值-工作量矩阵

```
高价值 │  ★ 优先做     │  ⚡ 需要规划
       │  (高价值低成本)  │  (高价值高成本)
       │               │
       ├───────────────┼───────────────
       │  ✓ 有空再做    │  ✗ 暂不考虑
低价值 │  (低价值低成本)  │  (低价值高成本)
       │               │
       └───────────────┴───────────────
              低工作量          高工作量
```

- [需求1]
- [需求2]

- [需求1]

```

## KANO 分析结果模板

```markdown
# KANO 模型分析结果

## 调研概况
- 调研时间：[时间段]
- 样本量：[N人]
- 目标用户：[用户描述]

## 分类结果

### 基本型需求（Must-be）
| 需求 | 满意度影响 | 优先级 |
|------|-----------|--------|
| [需求1] | 没有会强烈不满 | 最高 |

**策略**：必须100%满足，是产品底线

### 期望型需求（One-dimensional）
| 需求 | 满意度影响 | 优先级 |
|------|-----------|--------|
| [需求1] | 做得越好满意度越高 | 高 |

**策略**：根据资源尽量满足，体现产品竞争力

### 魅力型需求（Attractive）
| 需求 | 满意度影响 | 优先级 |
|------|-----------|--------|
| [需求1] | 有惊喜，没有也不会不满 | 中 |

**策略**：选择性投入，用于差异化

### 无差异型需求（Indifferent）
| 需求 | 说明 |
|------|------|
| [需求1] | 用户不关心 |

**策略**：不投入资源

### 反向型需求（Reverse）
| 需求 | 说明 |
|------|------|
| [需求1] | 做了反而会降低满意度 |

**策略**：避免开发

## 投资建议

按优先级分配资源：
1. 基本型需求 → 优先保障
2. 期望型需求 → 主要投入
3. 魅力型需求 → 差异化投入
4. 无差异型/反向型 → 不投入
```
