# 价值分析框架详解

## RICE 框架

### 完整评分指南

#### Reach（触达范围）

定义：在特定时间段内，这个需求会影响多少用户/客户？

评估方法：
- 使用产品数据估算影响用户数
- 统计时间周期：每月或每季度
- 考虑直接用户和间接用户

示例：
| 场景 | Reach 值 |
|------|----------|
| 影响所有用户的核心功能改进 | 10000（月活用户数） |
| 特定用户群的新功能 | 2000（目标用户数） |
| 后台管理功能 | 50（管理员数量） |

#### Impact（影响程度）

定义：对每个用户的影响有多大？

评分标准（固定值）：
- 3 = 巨大影响（改变用户行为/解决核心痛点）
- 2 = 高影响（显著改善体验）
- 1 = 中等影响（有帮助但不是必需）
- 0.5 = 低影响（轻微改善）
- 0.25 = 微小影响（几乎无感知）

#### Confidence（信心水平）

定义：对以上估算有多大把握？

评分标准：
- 100% = 高信心（有数据支撑/用户验证）
- 80% = 中等信心（有部分数据/类似案例）
- 50% = 低信心（主要靠直觉判断）

降低信心值的情况：
- 没有用户调研数据
- 是新领域/新尝试
- 技术方案不确定

#### Effort（工作量）

定义：需要多少人月来完成？

估算包含：
- 设计时间
- 开发时间
- 测试时间
- 上线部署时间

常见值：0.5、1、2、3、5、8、13

### RICE 计算示例

```
需求：添加社交分享功能

Reach = 5000（月活用户的 50%会使用）
Impact = 1（中等影响，方便用户分享）
Confidence = 80%（有竞品数据参考）
Effort = 2（约2人月）

RICE = (5000 × 1 × 0.8) / 2 = 2000
```

## MoSCoW 框架

### 分类决策树

```
这个需求...

没有它产品能否发布/正常使用？
├─ 不能 → Must Have
└─ 能 → 继续判断
    │
    对用户/业务价值高吗？
    ├─ 高，但有临时替代方案 → Should Have
    ├─ 中等，锦上添花 → Could Have
    └─ 低，或不适合现在做 → Won't Have
```

### 分类要点

**Must Have 判断标准**：
- 法规合规要求
- 核心业务流程必需
- 安全相关需求
- 没有则无法交付

**Should Have 与 Could Have 区分**：
- Should：影响用户体验或业务效率
- Could：增值功能，不做也不影响核心价值

**Won't Have 注意**：
- 不是"永远不做"，是"这次不做"
- 记录原因，便于后续版本考虑

## KANO 模型

### 用户调研方法

使用功能/反功能问卷：

**正向问题**：如果有这个功能，你感觉如何？
**反向问题**：如果没有这个功能，你感觉如何？

答案选项：
1. 我很喜欢
2. 理应如此
3. 无所谓
4. 勉强接受
5. 我很不喜欢

### KANO 评估矩阵

| | 没有-喜欢 | 没有-理应 | 没有-无所谓 | 没有-勉强 | 没有-不喜欢 |
|---|---|---|---|---|---|
| **有-喜欢** | 可疑 | 魅力型 | 魅力型 | 魅力型 | 期望型 |
| **有-理应** | 反向型 | 无差异 | 无差异 | 无差异 | 基本型 |
| **有-无所谓** | 反向型 | 无差异 | 无差异 | 无差异 | 基本型 |
| **有-勉强** | 反向型 | 无差异 | 无差异 | 无差异 | 基本型 |
| **有-不喜欢** | 反向型 | 反向型 | 反向型 | 反向型 | 可疑 |

### KANO 策略建议

| 类型 | 投资优先级 | 策略 |
|------|-----------|------|
| 基本型 | 最高 | 必须100%满足，但不要过度投入 |
| 期望型 | 高 | 投入越多满意度越高，找到平衡点 |
| 魅力型 | 中 | 选择性投入，用于差异化竞争 |
| 无差异型 | 低 | 不投入或最小化投入 |
| 反向型 | 无 | 避免开发 |

## ROI 分析

### 基本公式

```
ROI = (收益 - 成本) / 成本 × 100%
```

### 收益估算

**直接收益**：
- 新增收入
- 成本节约
- 效率提升的货币化价值

**间接收益**：
- 用户留存提升
- 品牌价值
- 降低风险

### 成本估算

**直接成本**：
- 开发人力成本
- 设计成本
- 测试成本
- 基础设施成本

**间接成本**：
- 机会成本
- 维护成本
- 培训成本

### ROI 计算示例

```
需求：自动化报表功能

收益估算：
- 每周节省10小时人工 × 52周 × 200元/小时 = 104,000元/年

成本估算：
- 开发成本：2人 × 1月 × 30,000 = 60,000元
- 维护成本：5,000元/年

第一年 ROI = (104,000 - 65,000) / 65,000 × 100% = 60%
```
